# Project Goal

## Decision

This repository's primary deliverable is **conference talk materials** for AWS Student Day 2026 ("Read Architecture with No Fear"). Everything else (demo repos, step-by-step docs, workflows) exists to support that talk.

- **Audience**: Mixed/general conference crowd (some non-technical attendees expected).
- **Role of the tooling walkthrough** (Terramaid + Terravision step docs): a **supporting segment** of the presentation, not the whole talk.
- **Demo scope**: the live demo focuses specifically on `tf-demo-repo` (not `tf-aws-account-storage`, which is optional/secondary).

## Why this matters

Because the repo's purpose is "supporting segment of a talk," documentation work (like the README) should be scoped to what's needed to follow/reproduce the demo — not to fully document every folder in the repo as if it were a standalone product.
